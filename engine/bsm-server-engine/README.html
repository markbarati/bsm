BSM Server Engine

نصب‌گر مرحله‌ای و قابل‌تکرار برای Ubuntu server، Ubuntu 24.04 ARM64، Hestia، Docker و Cloudflare
قابل ادامه
Hestia و Docker موجود را شناسایی می‌کند و از وضعیت نیمه‌نصب‌شده ادامه می‌دهد.
بدون حذف داده
/opt/appdata حذف نمی‌شود و Compose قبلی بکاپ می‌شود.
فقط 80/443
پنل‌های وب فقط روی localhost هستند و با Cloudflare Tunnel منتشر می‌شوند.
RDP انتخابی
حالت Cloudflare به پورت ورودی نیاز ندارد؛ 8888 فقط حالت اختیاری است.

اصلاحات نسخه دوم

اجرای تعاملی

tar -xzf bsm-server-engine.tar.gz
cd bsm-server-engine
sudo ./install.sh

اجرای فایل تنظیمات

cp config.env.example config.env
nano config.env
sudo ./install.sh --config ./config.env --non-interactive
اجرای مجدد پشتیبانی می‌شود. اسکریپت سرویس‌های انتخاب‌شده را reconcile می‌کند و داده برنامه‌ها را پاک نمی‌کند.

نمایش وضعیت

sudo ./scripts/status.sh

بررسی نهایی

sudo ./install.sh --verify

Cloudflare Tunnel

sudo cat /etc/bsm-engine/cloudflare-routes.md

تمام پنل‌های وب از بیرون روی HTTPS استاندارد 443 دیده می‌شوند، حتی اگر Origin داخلی پورت دیگری داشته باشد.

RDP پیشنهادی بدون پورت باز

rdp.example.com → RDP localhost:18888

روی ویندوز:

cloudflared access rdp --hostname rdp.example.com --url rdp://localhost:13389

سپس Remote Desktop را به localhost:13389 وصل کن.

حالت عمومی 8888

امنیت کمتری دارد. DNS باید خاکستری/DNS-only باشد و Oracle و Hestia فقط IP خانه را مجاز کنند.
RDP_EXPOSURE="public"
RDP_ALLOWED_CIDR="YOUR.PUBLIC.IP/32"

هشدار RDP

ایمیج LinuxServer rdesktop که Ubuntu Noble، ARM64، XFCE و xrdp را فراهم می‌کند deprecated است. بنابراین انتخابی است، Docker socket نمی‌گیرد و حالت پیش‌فرض آن Cloudflare است. Chromium با proot-apps به‌صورت persistent نصب می‌شود.
